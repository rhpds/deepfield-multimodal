= Demo Script
:toc: left
:toclevels: 3

This is a step-by-step script with exact clicks and expected results.
Use this when you need precision -- the xref:presenter-guide.adoc[Presenter Guide] covers what to say; this document covers what to click and what to expect.

== Pre-Demo Checklist

. Open the DeepField Multimodal URL in your browser
. Verify the app loads -- you should see the presentation view with a title slide
. If you want to show live LLM inference, confirm the Infrastructure panel (gear icon in the top right) shows "LLM inference via LiteLLM" with the model names
. If running CPU-only (default), confirm it shows "Rule-backed (no LLM configured)"
. Clear any previous demo state by refreshing the browser

== Part 1: Presentation

=== Step 1: Click Through Slides

*Click:* The right arrow or "Next" button to advance slides.

*Expected:* Each slide transitions with an animation. Seven slides total:

. Title -- DeepField Multimodal
. The Problem -- Signal overload
. The Thesis -- Classification before generation
. Three Tiers -- Nano, Micro, Macro diagram
. The Loop -- Signals, Decide, Act, Verify, Learn
. Intel + Red Hat -- CPU-native, no GPU
. Transition -- "Let me show you"

*Duration:* 4-5 minutes total for all slides.

=== Step 2: Start the Walkthrough

*Click:* The *Start Walkthrough* button on the final slide.

*Expected:* The UI transitions to the walkthrough view. The first act ("The Ordinary World") begins automatically.

== Fallback Procedures

=== LLM Timeout

*Symptom:* The demo hangs during micro or macro tier processing, or the Infrastructure panel shows errors.

*Fix:* The system has a 120-second timeout on LLM calls. If it times out, agents fall back to rule-backed classification automatically. You can also force rule-backed mode:

. Open browser developer console
. Run: `fetch('/api/v1/demo/stop', {method: 'POST'})`
. Refresh the page
. Restart the demo -- it will run in CPU-only mode

Alternatively, the demo works perfectly without any LLM configured. The walkthrough you see is the same whether LLM is connected or not.

=== No Events Appearing

*Symptom:* The evidence panel stays empty during Act 2.

*Fix:* Check that the fixture data exists:

. Open the Infrastructure panel (gear icon) and verify the agent list loads
. Try hitting the ingest endpoint directly: `curl -X POST <URL>/api/v1/demo/ingest`
. If fixture data is missing, the container may not have been built correctly -- check that `fixtures/` was included in the image

=== Demo Gets Stuck at a Pause Point

*Symptom:* Clicking Next does not advance.

*Fix:*

. Try clicking the Resume button (if the demo is paused rather than waiting for next)
. If that does not work, stop and restart: click Stop, then Start Walkthrough again
. As a last resort, refresh the browser and restart

=== Dashboard Shows "idle" State

*Symptom:* You clicked Start Walkthrough but nothing happens.

*Fix:*

. Check the browser console for errors
. Hit the state endpoint: `curl <URL>/api/v1/demo/state`
. If it returns `{"status": "idle"}`, the start request may not have reached the server -- try again
. Check that the backend pod is running: `oc get pods -n deepfield-multimodal`

=== Scale Run Takes Too Long

*Symptom:* The scale run acts take more than 30 seconds each.

*Fix:* This usually means LLM inference is being called during scale acts. The scale run should force rule-backed mode automatically. If it does not:

. The demo will still complete, just slower
. Mention to the audience: "We are using live LLM inference here to show the full capability -- in production, the scale tiers would run on CPU rules"
